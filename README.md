# SmartTask AI Advisor 🤖
### Advanced RAG & Event-Driven Analysis System

מערכת בינה מלאכותית מתקדמת לניתוח ותשאול תיעוד פרויקטים (Cursor, Claude, Kiro). המערכת מאחדת ידע ממקורות שונים לשכבת ידע אחת המאפשרת תשאול סמנטי ומבני בטכנולוגיית Event-Driven.


## 💡 סקירה כללית (Project Overview)
בסביבות פיתוח מודרניות, כלי AI מייצרים כמות אדירה של תיעוד הכולל החלטות, משימות ואזהרות המפוזרים בקבצי `.md` שונים. המערכת שפיתחתי משמשת כ**"מוח מרכזי"** שסורק את כלל התיעוד ומנגיש אותו למפתח בממשק אחד חכם. המערכת יודעת להבחין בין שאלות של **"הקשר"** (למה בחרנו בטכנולוגיה מסוימת?) לבין שאלות של **"נתונים קשיחים"** (כמו אזהרות).

---

## 🚀 שלבי פיתוח והרצה (Step-by-Step Guide)

### 1️⃣ שלב א' - MVP: חיפוש סמנטי
הקמת תשתית ה-RAG הראשונית הכוללת יצירת Embeddings בעזרת Cohere וחיבור למסד נתונים וקטורי Pinecone.  
**פקודת הרצה:** `python step1_mvp.py`

### 2️⃣ שלב ב' - ארכיטקטורת Event-Driven Workflow
שכתוב המערכת למבנה מבוסס אירועים (Events). כל פעולה מוגדרת כצעד (Step) נפרד בתוך Workflow, מה שמאפשר ניהול מצב וולידציות על קלט המשתמש.  
**פקודת הרצה:** `python step2_workflow.py`

### 3️⃣ שלב ג' - חילוץ נתונים מובנים (Data Extraction)
סריקה חכמה של הקבצים וחילוץ החלטות טכניות, כללי פיתוח ואזהרות סיכון לתוך קובץ JSON מובנה, כולל Metadata מלא של מיקום המידע.  
**פקודת הרצה:** `python step3_extraction_full.py`

### 4️⃣ שלב ד' - המערכת הסופית (The Smart Agent)
חיבור כל הרכיבים עם נתב (Router) חכם המנתב בין שליפה מה-JSON לשליפה מה-Vector DB בהתאם לכוונת המשתמש.  
**פקודת הרצה:** `python final_app_stage3.py`

## 🧪 בדיקות קבלה (QA) - דוגמאות הרצה

להלן השאלות שנבדקו במערכת והתוצאות המדויקות שהתקבלו:

### 📍 בדיקות שליפה מובנית (Structured Path)
* **שאלה:** "אילו אזהרות קיימות בפרויקט?"
    * **מקור מידע:** `structured_data_full.json`
    * **תוצאה:** רשימה של 5 אזהרות הכוללת רמת חומרה והפניה לשורה מדויקת עם סימון המיקום [📍].
* **שאלה:** "מהן החלטות ה-Backend שהתקבלו?"
    * **מקור מידע:** `structured_data_full.json`
    * **תוצאה:** זיהוי הבחירה ב-FastAPI ו-Python כולל הפניה לקובץ `dev_plan.md` בשורה 12.

### 🌲 בדיקות חיפוש סמנטי (Semantic Path)
* **שאלה:** "איך המערכת מטפלת בעומסי עבודה גבוהים?"
    * **מקור מידע:** `Pinecone Vector Index`
    * **תוצאה:** הסבר מפורט על שימוש ב-CDN, Caching וארכיטקטורת מיקרו-שירותים.
* **שאלה:** "מהם עקרונות אבטחת המידע במערכת?"
    * **מקור מידע:** `Pinecone Vector Index`
    * **תוצאה:** פירוט טכני על מנגנוני הצפנה (bcrypt), שימוש ב-JWT וניהול הרשאות RBAC.

## 🧠 רפלקציה ותובנות

* **השילוב המנצח:** גיליתי שחיפוש סמנטי לבדו עלול "להמציא" נתונים כשהוא סורק טבלאות. הוספת ה-Structured Extraction הבטיחה אמינות של 100% בשליפת נתונים קשיחים כמו אזהרות והחלטות טכנולוגיות.
* **ניתוב חכם:** השימוש ב-Router מאפשר למערכת להיות חסכונית - היא ניגשת ל-Vector DB רק כשבאמת צריך הקשר רחב, ושולפת מהר מה-JSON כשמדובר בשאילתות סטטוס.
* **דיוק במקורות:** כל תשובה מובנית כוללת את ה-Line Range וה-Anchor, מה שמאפשר למפתח לחזור לקוד המקור בשניות.
